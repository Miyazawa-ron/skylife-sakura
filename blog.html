When You Have Your First AI Employee, How Do You Manage It? 最初のAI社員ができたら、どう管理するのか？
April 18, 2026 · 12 min read 2026年4月18日 · 読了12分
↓

Most people feel surprised the first time they use an AI Agent, confused the second time, and chaotic after the third.

The surprise comes from efficiency. The confusion comes from it sometimes working, sometimes not. The chaos comes from running three or four AI tools at the same time — each operating independently, unaware of each other, with no coordination. The entire workflow becomes harder to maintain than before.

This is not a problem with AI itself. It is a governance problem.

Recently I studied a GitHub repository: Hermes Agent Control Room. It is not an AI application itself, nor a new model, but a methodology and documentation system for managing and orchestrating teams of AI Agents. After reading it, I realized this approach deserves a serious article.

Where Does the Problem Lie?

Let me pour some cold water: most people who claim to be "working with AI" are essentially just chatting with AI.

The difference between chatting and working is that work requires memory, role division, process衔接, and traceable records when problems arise. When every AI conversation starts from zero, when different Agents don't know each other, and when there is no documentation recording "what this AI is responsible for", what you have is not a team — but a group of forgetful temporary workers.

"Build the Control Room first, then plug in Agents. Avoid turning AI into a bunch of scattered robots."

This sentence seems simple, but the logic behind it is worth pondering: it analogizes the management of AI Agents to human resource management in a company. You wouldn't hire a bunch of employees and let them coordinate themselves. You need job responsibilities, reporting chains, and task distribution systems.

What is Control Room?

Control Room is not where AI runs, but where AI is described, regulated, and governed. In technical terms, it is a "control plane" separated from the actual running Agent instances.

For example: Control Room is like a company's HR system and internal Wiki — it records who everyone is, what they are responsible for, what rules exist, where passwords are stored, and who to contact when problems arise. But it doesn't do the work itself. The Agents below do the actual work.

Four Growth Levels

One of the most valuable parts of this system is that it defines a growth path from "personal toy" to "team infrastructure", divided into four levels, each with clear applicable scenarios.

For myself, I am currently around Level 1.5 — multiple tools are running, but roles are vague and there is no documentation. This grading made me realize what I should do next.

Task Bus: Enabling Agents to "Talk" to Each Other

The core problem of multi-Agent collaboration is communication: when one Agent completes a task, how does the result get passed to the next?

The solution in this repository is very engineering-oriented and simple — using file directories to simulate a message queue. Each Agent has four folders: inbox, working, outbox, and archive.

The elegance of this design lies in the fact that it does not rely on any middleware or third-party services. The file system itself becomes the message queue. It is simple, auditable, and when problems occur, you can open the folder to see the status.

Security: The Easiest Rule to Break

In multi-Agent scenarios, security is a real issue, not a theoretical risk. When you have four or five Agents sharing API Keys, and keys are scattered across configuration files, the probability of leakage or misuse rises rapidly.

This repository proposes a separation principle: Record the name and location of keys, never record the value of the keys.

The value of this rule is that the Control Room documentation can be discussed publicly, uploaded to Git, and collaborated on by multiple people — because it only contains structure, not secrets.

8 Built-in Skills, Ready to Use

In addition to the methodology, this repository also includes 8 reusable "Skills" — each Skill is a standard operating manual that tells Agents how to complete specific tasks.

My Inspiration and Calm Assessment

After reading this repository, my first reaction was excitement, and my second reaction was restraint.

The reason for excitement: this approach helped me clearly see the root of chaos in my own workflow — I had tools, but no governance. Each AI tool operated independently, with no role definitions, no documentation, and no unified task tracking.

The reason for restraint: this repository currently has only 7 commits. There is a clear gap between the 248 stars and the actual completion level of the repository. It is more like a carefully designed blueprint than a system that has been validated in large-scale production. The entire architecture assumes you are running Ubuntu/Debian VPS, and is not directly applicable to macOS local environments.

多くの人がAI Agentを初めて使ったときの感想は「驚き」、2回目は「混乱」、3回目以降は「カオス」になる。

驚きは効率から来る。混乱は「時々うまくいくが、時々うまくいかない」ことから来る。カオスは、3〜4つのAIツールを同時に動かしているときに起こる。それぞれが独立して動き、互いを知らず、調整もないため、全体のワークフローが以前より管理しにくくなる。

これはAI自体の問題ではない。これはガバナンスの問題である。

最近、私はあるGitHubリポジトリを研究した：Hermes Agent Control Room。これはAIアプリケーションそのものではなく、新しいモデルでもない。AI Agentチームを管理・編成するための方法論とドキュメント体系である。これを読んで、私はこのアプローチを真剣に記事にすべきだと感じた。

問題はどこにあるのか？

まず冷や水を浴びせよう。「AIで仕事をしている」と主張する人のほとんどは、実際にはAIと「チャット」しているだけだ。

チャットと仕事の違いは、仕事には記憶、役割分担、プロセスのつながり、そして問題が発生したときに追跡可能な記録が必要だということだ。AIとの会話が毎回ゼロから始まり、異なるAgent同士が互いを知らず、「このAIは何を担当しているか」というドキュメントもない場合、あなたが持っているのはチームではなく、忘れっぽい派遣社員の集まりである。

「まずControl Roomを構築し、それからAgentを接続せよ。AIを散らばったロボットの集まりにしないために。」

この文はシンプルに見えるが、その背後にある論理は深く考える価値がある。これはAI Agentの管理問題を、企業の人的資源管理に例えている。あなたは従業員を大勢雇ってから彼らに勝手に調整させたりはしないはずだ。職務記述書、報告ライン、タスク配分システムが必要になる。

Control Roomとは何か？

Control RoomはAIが実際に動く場所ではない。AIを記述し、規制し、統治する場所である。技術的な言葉で言えば、それは実際のAgentインスタンスから分離された「コントロールプレーン」である。

例えるなら、Control Roomは会社のHRシステムや社内Wikiのようなものだ——そこには「誰が誰で、何を担当していて、どんなルールがあり、パスワードはどこにあり、問題が起きたら誰に連絡するか」が記録されている。しかしControl Room自体は仕事はしない。実際に仕事をするのは下のAgentたちである。

4つの成長レベル

このシステムのもっとも価値のある部分の一つは、「個人の玩具」から「チームのインフラ」への成長パスを定義していることだ。4つのレベルに分かれており、それぞれに明確な適用シーンがある。

私自身は現在、Level 1.5くらいの位置にいる。複数のツールは動いているが、役割が曖昧で、ドキュメントもない。この分類を見て、私は次に何をすべきかが明確になった。

Task Bus：Agent同士が「話せる」ようにする

マルチAgent協調の核心的な問題はコミュニケーションである。あるAgentがタスクを完了したとき、その結果を次のAgentにどう渡すのか？

このリポジトリの解決策は非常に工学的で、かつシンプルだ——ファイルディレクトリを使ってメッセージキューをシミュレートする。各Agentはinbox、working、outbox、archiveの4つのフォルダを持つ。

この設計の優れた点は、ミドルウェアやサードパーティサービスに依存しないことだ。ファイルシステム自体がメッセージキューになる。シンプルで、監査可能で、問題が発生したらフォルダを開けば状態がわかる。

セキュリティ：最も犯しやすい鉄則

マルチAgentのシナリオでは、セキュリティは現実的な問題であり、理論上のリスクではない。4〜5つのAgentがAPI Keyを共有し、鍵がさまざまな設定ファイルに散らばっている場合、漏洩や誤用の確率は急速に上昇する。

このリポジトリは一つの分離原則を提案している：鍵の名前と場所は記録するが、鍵の値自体は絶対に記録しない。

このルールの価値は、Control Roomのドキュメントを公開で議論したり、Gitに上げたり、複数人で共同編集したりできることにある。なぜなら、そこには構造だけがあり、秘密はないからだ。

8つのビルトインSkill、すぐに使える

方法論に加えて、このリポジトリには8つの再利用可能な「Skill」が組み込まれている。各Skillは特定のタスクをAgentが完了するための標準的な操作マニュアルである。

私自身のインスピレーションと冷静な評価

このリポジトリを読んだ後、私の最初の反応は興奮、2番目の反応は抑制だった。

興奮の理由：このアプローチは、私自身のワークフローのカオスな根本原因を明確に示してくれた。ツールはあったが、ガバナンスがなかった。各AIツールが独立して動き、役割定義もドキュメントもなく、統一されたタスク追跡もなかった。

抑制の理由：このリポジトリは現在、コミット数が7回しかない。248スターの人気と、リポジトリ自体の完成度との間には明らかなギャップがある。これは大規模な本番環境で検証されたシステムというより、丁寧に設計された青写真に近い。全体のアーキテクチャはUbuntu/Debian VPSで動作することを前提としており、macOSローカル環境には直接適用できない。

Japan's Business Manager Visa Applications Plummeted 96%: How It Happened and Who Got Hurt 経営・管理ビザの申請数が96%減少：日本はどうやって実現したのか、そして誰が傷ついたのか
May 18, 2026 · 18 min read 2026年5月18日 · 読了18分
↓

In October 2025, Japan quietly changed one immigration rule.

Half a year later, the number of new applications for the Business Manager visa had dropped by 96%.

This figure was exclusively obtained by TBS News23 reporters from sources within the Ministry of Justice in May this year. Before that, many people were not even aware that this had happened.

This article takes a closer look at how it occurred, what mechanisms were at work behind it, and what it means for foreign business owners living in Japan.

The Starting Point Was Not the Government, But Members of Parliament

Many believe Japanese policy is decided solely by the government. In reality, policy changes are often triggered during parliamentary questioning.

In spring 2025, during a Budget Committee session in the House of Councillors, a member of the Japan Innovation Party publicly stated:

"The Business Manager visa is very well-known in China. There are specialized intermediaries online promoting it: by opening a company in Japan, one can enjoy high-cost medical care almost for free. Foreigners only need to stay in Japan for three months to join the National Health Insurance; once the company is established, they can immediately join association health insurance and access high-cost medical expense systems. This is a loophole in the system."

Then-Prime Minister Shigeru Ishiba responded: "The Japanese people have long cherished the spirit of social solidarity and mutual assistance. Exploiting this virtue to game the system is absolutely not permitted."

This parliamentary exchange sent a clear political signal to the Ministry of Justice: it was time to act.

How the Administration Turned Political Will into Regulation

Once the political signal was given, the Immigration Services Agency under the Ministry of Justice began the reform process.

One important detail: in Japan, amending a ministerial ordinance does not require a Diet vote, but it must go through a public comment period (パブリックコメント) — soliciting opinions from the public and providing written responses to feedback.

On October 16, 2025, the new rules officially took effect. The key changes were:

  • Capital requirement: raised from ¥5 million to ¥30 million
  • Full-time employees: must have at least one (mandatory)
  • Manager qualifications: now require at least three years of experience or a master's degree or higher
  • Business plan: must be verified by a certified public accountant or small and medium enterprise diagnostician

From parliamentary questioning to regulatory implementation took less than half a year. The execution speed of Japanese administrative reform can sometimes leave people unprepared.

Immigration Enforcement: Step Out in the Morning, and They Already Know Your Details by Afternoon

After the policy was implemented, the Immigration Services Agency immediately began on-site inspections. TBS reporters captured one such scene: around 9 a.m., immigration officers in Tokyo got into cars and headed to a company for an on-site verification.

The agency has significant investigative powers. They can demand to review:

  • Business license and corporate registration
  • Financial statements and proof of capital payment
  • Employee labor contracts and social insurance payment records
  • Actual office usage (with photos for records)
  • Whether the corporate address and residential address are separate

There have already been real cases: a restaurant was denied visa renewal because the kitchen was on the first floor and the family residence on the second, which was deemed "business premises mixed with residence."

What Role Did the Media Play?

TBS News23's reporting focused on a specific individual: Gopal (50), from Ichikawa City, Chiba Prefecture. Originally from Nepal, he came to Japan 19 years ago as a chef. After three years of training, he obtained a Business Manager visa and now runs two Indian-Nepalese curry restaurants. He pays taxes and insurance on time and is well-liked by local regular customers.

His current capital: ¥5 million. He is short ¥25 million from the new ¥30 million requirement.

"I pay insurance and taxes, and have lived in Japan for a long time. But ¥30 million... honestly, it's really difficult," he said.

After the broadcast, public opinion began to focus on the "one-size-fits-all" problem. Legal scholars, administrative scriveners, and support groups for foreigners in Japan started voicing concerns: Is the capital threshold too high? Should more nuanced evaluation criteria be introduced?

How the Separation of Powers Actually Works Here

Textbooks tell us that legislative, executive, and judicial powers are separated and check each other. In this case, the actual operational logic was as follows:

  • Diet (Legislature): Triggers policy signals through questioning, but does not directly legislate.
  • Cabinet & Ministry of Justice (Executive): Implements changes quickly through ministerial ordinance amendments without Diet voting.
  • Immigration Agency (Enforcement): Conducts on-site inspections with considerable discretionary power.
  • Judiciary: In theory, administrative lawsuits are possible, but success rates are historically low and costs are high.
  • Media & Public: Public pressure is the most active external check in reality.

A structural issue worth noting: because ministerial ordinance amendments do not require Diet approval, as long as the Cabinet and Ministry of Justice are aligned, policies can be completed within months. This is both efficient and risky, as it bypasses direct democratic deliberation.

Who Got Hurt, and Who Did Not?

Behind the 96% drop, two groups were affected differently, even though the policy text itself makes no distinction:

  • The group that was rightly removed: Those who exploited the system through paper companies. This was the real target of the reform.
  • The group that was collateral damage: People like Gopal — rooted in Japan for nearly 20 years, law-abiding, but lacking the new ¥30 million capital. The one-size-fits-all standard lumped them together with the previous group.

This is the most debatable aspect of the reform and the central tension in the media coverage. A source close to the Ministry of Justice noted that most newly approved Business Manager visa holders after the tightening "are mostly executives of listed companies."

Practical Advice for Foreign Business Owners in Japan

For foreigners holding Business Manager visas and operating in Japan, this political-legal mechanism sends a relatively clear signal: Rules can change, but changes are traceable and come with signals.

  • Pay attention to Diet committee questioning related to residence status — the spring 2025 debates already signaled the direction of reform.
  • Regularly check the Immigration Services Agency website for public comment periods on ordinance amendments.
  • Conduct an annual "residence status health check" with an administrative scrivener.
  • Maintain proper accounting, insurance, and employee contracts — this remains the best protection under any tightening.
  • Clearly separate business premises from residence (including address registration and actual usage).

Japan's system is not perfect, but it is transparent and traceable. When operating here, the most important thing is to read the signals of rule changes, rather than panicking only after the rules have already fallen.

2025年10月、日本は移民ルールを静かに変更した。

半年後、経営・管理ビザの新規申請数は96%減少した。

この数字は、今年5月にTBS News23の記者が法務省関係者から独占取材したものだ。それまで、多くの人はこの変化がすでに起こっていたことすら知らなかった。

この記事では、それがどのように起こったのか、背後でどのような仕組みが働いていたのか、そして在日外国人経営者にとって何を意味するのかを丁寧に考察する。

起点は政府ではなく、国会議員だった

多くの人が「日本の政策は政府が決める」と思っているが、実際には政策変更の引き金は国会の質疑にあることが多い。

2025年春、参議院予算委員会で、日本維新の会の議員が公にこう述べた：

「経営・管理ビザは中国で非常に有名だ。ネットには『日本で会社を作れば、高額医療をほぼ無料で享受できる』と宣伝する専門の仲介業者がいる。外国人は日本に3ヶ月滞在するだけで国民健康保険に入れ、会社が設立されればすぐに協会けんぽに加入でき、高額療養費制度も利用可能。これは制度の抜け穴だ。」

当時の石破茂首相はこう答えた。「日本人は長年、社会連帯と相互扶助の精神を大切にしてきた。この美徳を利用して制度を悪用することは、絶対に許されない。」

この国会答弁は、法務省に明確な政治的シグナルを送った。「動ける」というメッセージだった。

行政はどのように「政治的意思」を法規に変えたのか

政治的シグナルが出された後、法務省出入国在留管理庁（入管庁）は改革手続きを開始した。

ここで注目すべき点がある。日本で「省令」を改正する場合、国会の採決は不要だが、パブリックコメント（公募意見）の手続きを経なければならない。社会に意見を公開募集し、フィードバックに対して書面で回答する。

2025年10月16日、新ルールが正式に施行された。主な変更点は以下の通り：

  • 資本金要件：500万円 → 3,000万円
  • 常勤従業員：1名以上（必須）
  • 経営者資格：3年以上の経験または修士以上の学位
  • 事業計画：公認会計士または中小企業診断士による確認が必要

国会質疑から法規施行まで、半年もかからなかった。日本行政改革の実行速度は、時に人々が準備する暇を与えないほど速い。

入管の執行力：朝出かければ、午後にはあなたの底が知れている

政策施行後、入管庁はすぐに実地調査を開始した。TBSの記者が撮影したある場面では、朝9時過ぎに東京入管の職員が車に乗り込み、ある会社へ実地検証に向かっていた。

入管庁の調査権限はかなり大きい。以下のような書類の提出を要求できる：

  • 営業許可証と法人登記
  • 財務諸表と資本金払込証明
  • 従業員の労働契約と社会保険納付記録
  • 事務所の実際の使用状況（写真撮影・記録）
  • 法人住所と居住住所が分離されているか

すでに実例がある。ある飲食店は、1階が厨房、2階が家族の居住スペースだったため「事業所と住所の混同」と認定され、ビザ更新を拒否された。

メディアはここで何をしたのか

TBS News23の報道は、具体的な一人の人物に焦点を当てた。千葉県市川市在住のゴパルさん（50歳）。ネパール出身で19年前にコックとして来日し、3年間の修業後に経営・管理ビザを取得。現在はインド・ネパールカレーの店を2店舗経営し、税金と保険をきちんと納め、地元の常連客に愛されている。

現在の資本金：500万円。新要件の3,000万円まで、あと2,500万円足りない。

「保険も税金も払っているし、日本で長く暮らしてきた。でも3,000万円は、正直本当に厳しい」と彼は語った。

この報道が放送された後、世論は「一律適用」の問題に注目し始めた。法律学者、行政書士、在日外国人支援団体などが声を上げ、「資本金要件は高すぎないか？より細やかな評価基準を設けるべきではないか？」と問いかけた。

三権分立はここでどのように機能しているのか

教科書では「立法・行政・司法の三権分立が相互に抑制する」と教わる。この件での実際の運用ロジックは以下の通りだった：

  • 国会（立法）：質疑を通じて政策シグナルを出すが、直接立法はしない。
  • 内閣・法務省（行政）：省令改正により迅速に施行（国会採決不要）。
  • 入管庁（行政執行）：実地調査を行い、かなり大きな裁量権を持つ。
  • 司法：理論上は行政訴訟可能だが、勝率は低くコストが高い。
  • メディア・世論：現実において最も活発な外部抑制力。

ここで注目すべき構造的問題がある。省令改正は国会採決を必要としないという点だ。つまり、内閣と法務省の意思が一致すれば、数ヶ月で政策を完了させられる。これは効率であると同時にリスクでもある。最も直接的な民主的審議手続きを迂回しているからだ。

今回は誰が傷つき、誰は傷つかなかったのか

96%という減少数字の背後には、政策条文自体は区別していないにもかかわらず、実際には二つのグループが異なる影響を受けた。

  • 当然排除されるべきグループ：ペーパーカンパニーで制度の抜け穴を悪用していた人々。これが今回の改革の本当のターゲットだった。
  • 巻き添えを食らったグループ：ゴパルさんのような人々。日本に20年近く根を張り、法を守りながら生活してきたが、資本金が3,000万円に届かない。一律適用により、彼らは前者のグループと同じ枠組みに押し込められた。

これが今回の改革で最も議論を呼んでいる点であり、メディア報道の核心的な緊張関係でもある。法務省関係者は、厳格化後に新たに許可された経営・管理ビザ保有者の多くが「上場企業の役員クラス」であると述べた。

在日経営者への実務的アドバイス

経営・管理ビザを持ち日本で事業を営む外国人にとって、この政治・法制度は比較的明確なシグナルを送っている。ルールは変わるが、変化は追跡可能であり、兆候がある。

  • 国会委員会で在留資格に関する質疑に注目する。2025年春の国会論戦は、すでに改革の方向性を示していた。
  • 入管庁のウェブサイトで省令改正に関するパブリックコメント募集を定期的に確認する。
  • 行政書士と年に一度、在留資格の「健康診断」を行う。
  • 会計、保険、従業員契約の規律を保つ。これはどんな政策 tightening でも有効な護身符となる。
  • 事業所と住所を明確に分離する（住所登記と実際の使用状況を含む）。

日本の制度は完璧ではないが、透明性があり、追跡可能だ。ここで事業を営む上で最も重要なのは、ルール変更のシグナルを読み取ることだ。ルールが降りかかってきてから慌て始めるのではなく。
